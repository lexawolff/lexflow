# Padrões do Projeto

## Filosofia

Done is better than perfect.

Entregar valor continuamente.

Não voltar atrás sem necessidade real.

---

## Arquitetura

A arquitetura atual está congelada.

Nenhuma refatoração estrutural será feita sem necessidade técnica comprovada.

---

## Desenvolvimento

Cada Sprint possui objetivo definido.

Cada entrega deve funcionar antes da próxima começar.

Nenhuma funcionalidade será iniciada com erros pendentes.

---

## Interface

Visual limpo.

Poucos cliques.

Poucos campos obrigatórios.

Tema centralizado.

Identidade visual consistente.

---

## Produto

Toda funcionalidade deve responder à pergunta:

"Ela reduz trabalho do advogado?"

Se a resposta for não, ela não é prioridade.

---

## IA

A IA nunca substitui a decisão jurídica.

Ela auxilia.

Sempre apresenta fundamentos.

Sempre mostra as fontes.

Sempre permite revisão pelo advogado.
